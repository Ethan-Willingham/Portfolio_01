The Box Shadow Sponge

23rd of April, 2026

David Gerrells' How Not to Use Box Shadows escalates from CSS experiments to a full ray tracer — all rendered through box-shadow on a single <div>. One demo stopped me cold: the sponge. Colored circles that drift to home positions. Pull them apart with your cursor. Release. They drift back. I rebuilt it to understand how it works.

No <canvas>. No WebGL. No SVG. Every circle is a box-shadow value on a 1×1 pixel element.

• • •

How it works

box-shadow accepts a comma-separated list of shadows. Each takes an x offset, y offset, blur, spread, and color. Set blur to zero, give it a spread, put it on a 1px element with border-radius: 50% — you get a hard circle at arbitrary coordinates. Stack 120 and you have a particle system.

The render function runs every frame. Sort by depth, compute spread, join into a string, set it on the element:

function render() {
  balls.sort(function (a, b) { return b.z - a.z; });
  var parts = [];
  for (var i = 0; i < balls.length; i++) {
    var b = balls[i];
    var spread = (b.size * (1 + b.z / 40) - 1) / 2;
    parts.push(b.x+'px '+b.y+'px 0 '+spread+'px '+b.color);
  }
  renderDot.style.boxShadow = parts.join(',');
}

The sponge effect

Each ball has a home position. A spring force pulls it back. When the pointer is active, nearby balls get yanked toward it instead. The pull overwhelms the spring — particles rip away. Release, and the springs take over:

// Spring toward home
b.vx += (b.homeX - b.x) * springK * dt;
b.vy += (b.homeY - b.y) * springK * dt;

// Pointer pull (within radius)
var f = (1 - dist / 250) * 1800;
b.vx += (px / dist) * f * dt;

// Damping bleeds off energy
b.vx *= 0.91;

Fake 3D and performance

Each ball's z value scales its visual size via 1 + z / 40. Sorting by z gives natural occlusion. It's not real perspective but it's enough to feel organic.

It's fast because there's no blur and no transparency. Box-shadows map to the same rounded-rectangle primitives GPUs already use for UI compositing. Apple Silicon pushes thousands at 60fps. The real bottleneck is string concatenation — building a ~5KB shadow string every frame.

Gerrells takes this all the way to a progressive ray tracer with specular reflections and emissive lighting — entirely through box-shadow. Full article and source at dgerrells.com.
