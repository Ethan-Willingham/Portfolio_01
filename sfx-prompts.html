Sluice SFX Prompts

Every sound-effect prompt, one tap to copy. Mirrors SFX_PROMPT_SYSTEM.md §5. The companion design doc is SFX_BIBLE.md.

1 Tap Copy on a sound and paste the prompt into the tool shown by its badge: ElevenLabs for one-shots and short loops, Stable Audio for the ambience beds.

2 Set the Duration / Loop / Influence shown on the card. For Stable Audio beds, also copy the Negative field into its negative-prompt box.

3 Generate 5 to 8, keep the best (or the best few where it says pool N). Audition candidates over the real music in the SFX test bench; loops get a seamless crossfade + WAV download there.

4 Render mono, convert (afconvert -f m4af -d aac in.wav out.m4a), save as the exact Save as filename on the card, drop it in assets/sfx/. Every card's key + call site is wired in-game: each saved file lights up on the next reload.

Positive description only for ElevenLabs (it has no negative field); the Negative line is for Stable Audio, which does. Keep everything mono and dry for Effect sounds (the game adds depth filtering and reverb live). Tonal layers (the ore-pickup chime, the discovery chord) live in the music system tuned to D minor, so the SFX here only carry the noise/transient layer.

Already synthesized live in js/audio.js (no need to generate): the whole flight pack, that is the wind bed, engine growl, stall horn, vario, ignition, sonic boom, vapor cone and touchdown thumps. The jetpack-* and land-* cards are optional asset upgrades over that synth. The combat, rig-drive, jet-spin and air-pulse keys are wired too (2026-06-10): every card on this page lights up the moment its file lands in assets/sfx/.

Verified against the ElevenLabs and Stable Audio docs (v1.1). Full rationale in SFX_BIBLE.md and SFX_PROMPT_SYSTEM.md.
